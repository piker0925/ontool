# ADR-0012 댓글 무인증 단순 구현

## 상태
확정

## 배경
각 도구 페이지에 피드백/건의사항을 남길 수 있는 댓글 기능을 추가하기로 했다.
이 서비스에는 로그인이 없다.

## 검토한 선택지

| 방식 | 설명 | 문제점 |
|------|------|------|
| A. 로그인 필수 | 댓글 작성 시 회원가입 요구 | 로그인 시스템 구현 필요. 사용 허들 높아짐 |
| B. 익명 + 스팸 방지 | reCAPTCHA, rate limiting 등 | 구현 복잡도 증가. 외부 서비스 의존 |
| C. 완전 오픈 | 누구나 인증 없이 작성 가능 | 스팸 가능성 있음 |
| D. 관리자 삭제 기능 포함 오픈 | C + 관리자가 삭제 가능 | 스팸 대응 수단 확보 |

## 결정
**D. 완전 오픈 + 관리자 삭제**

- 댓글 작성: 인증 없음. content만 입력.
- 관리자는 `/admin/comments/{id}` DELETE로 댓글 삭제 가능.
- 스팸 방지 로직 없음.

## 이유

이 서비스의 실제 방문자는 소수(포트폴리오 열람자, 지인)다.
스팸이 대규모로 발생할 확률이 극히 낮다.
reCAPTCHA나 rate limiting은 구현 비용 대비 효과가 없는 과잉 설계다.

스팸이 실제로 발생하면 관리자 삭제 기능으로 수동 대응한다.
서비스 규모가 커지면 그 시점에 추가한다.

## 결과
- Comment 테이블: id, module_id, content, created_at (작성자 정보 없음)
- `POST /api/v1/tools/{moduleId}/comments` — 인증 없음
- `DELETE /admin/comments/{id}` — HTTP Basic Auth 필요
- 닉네임·이메일 등 개인정보 수집 없음
