# ADR-0023 종합 도구 포털 전환 — 랜딩 대문 + 4구역 IA

## 상태

확정 (2026-07-17) — 구현 전 (v2 1차). ADR-0015/0018의 "평면 8카테고리 + 런처형 홈" IA를 대체·확장한다.

> 후속(같은 날): 본문의 "브랜드 점진 전환" 중 ②새 이름이 **OnTool/온툴**로 확정됐다(표기·워드마크·파비콘·슬로건은 DESIGN.md §0, 적용은 이슈 044·045). GitHub 레포명도 `ontool`로 변경 완료. ④도메인 구매만 보류로 남음.

## 배경

프로젝트를 주도하는 이해관계자(강사)의 요구로 기능 대량 확장이 확정됐다(`.scratch/v2-feature-list.md`(당시 `module-ideas-brainstorm.md`, 후보 60여 개 — 이후 통합·폐기) — 개발자 도구 외에 일반인 생활 계산기·재미 도구 포함). 이 규모에서 두 가지 문제가 생긴다:

1. **IA 한계**: 현재 사이드바의 평면 8카테고리는 도구 60개+를 못 견딘다.
2. **정체성 충돌**: SQL 포맷터와 출산예정일 계산기가 한 목록에 섞이면 "개발자 도구 사이트"라는 정체성이 흐려지고, 포트폴리오 관점에서도 제품 경계 판단력에 대한 의문을 남긴다.

일반인용 도구를 별도 사이트로 분리하는 안은 "이 프로젝트가 많은 기능을 제공해야 한다"는 요구와 충돌하고, 배포·공유 컴포넌트 유지비와 검색/즐겨찾기 단절을 만든다. → **물리 분리 대신 정보 구조(IA) 계층에서 분리한다.**

## 결정

**사이트를 "랜딩 대문 + 대상별 4구역" 허브 구조의 종합 도구 포털로 전환한다.**

- **`/` = 랜딩 대문.** 사이드바 없는 전용 레이아웃. 첫 방문자에게는 구역 선택 허브(멋진 첫인상), 재방문자에게는 ⌘K 검색·최근 사용·즐겨찾기로 도구까지 원스텝 진입을 제공한다. 구역 진입 후에는 대문을 다시 거치지 않는 동선.
- **4구역** (같은 Vue 앱 안의 라우트 네임스페이스 — 별도 앱/도메인 아님):

  | 라우트 | 표시명 | 내용 |
  |--------|--------|------|
  | `/dev` | 개발자 도구 | 포맷터·인코더·데이터 변환·코드생성·보안·네트워크·DevOps |
  | `/files` | 파일·문서 | 이미지·PDF·오피스 (대상 무관 공용 — 잠재 수요 최대 묶음) |
  | `/life` | 생활 도구 | 계산기류 (연봉·대출·날짜·단위 등) |
  | `/fun` | 재미·게임 | 로또·사다리·닉네임·워드클라우드 등 + 미니게임 |

- 구역마다 **액센트 컬러·사이드바 구성을 달리해** "다른 사이트에 온 느낌"을 디자인으로 낸다. 기존 런처형 HomePage의 역할(즐겨찾기/최근/카테고리 섹션)은 각 구역 홈이 승계한다.
- **도구 상세 URL은 `/tools/:id` 유지.** 구역은 도구의 소속이 아니라 탐색 컨텍스트다 — 한 도구가 복수 구역에 노출될 수 있다(예: EXIF 제거기는 `/dev`·`/files` 양쪽). 기존 북마크·검색 유입·`moduleId` 참조(좋아요·댓글)가 전부 생존하고 리다이렉트 공사가 없다.
- **브랜드 점진 전환**: "DevToolbox"는 개발자 전용을 함의하므로 이름 교체를 전제한다. 단계: ① 1차 공사에서 사이트 표시명을 상수 한 곳으로 모음 → ② 새 이름 확정(별도 진행) → ③ 표시명·로고 교체 → ④ (선택) 도메인 구매 — 구매 시점에 쿠키 인증 전환(ADR-0024) 재검토 여지.

## 기각한 대안

- **별도 사이트 분리** — 요구(단일 프로젝트의 기능 규모)와 충돌, 인프라 이중화.
- **구역 하위 URL**(`/dev/tools/json-formatter`) — 기존 URL 전부 리다이렉트 필요, 도구가 한 구역에만 소속돼 복수 노출 불가, 정본 URL 애매.
- **3구역**(개발자/비개발자/재미) — 이미지·PDF가 개발자 구역에 갇혀 일반인 동선이 막히고 구역 크기 불균형이 심함.
- **5구역+** — 대문은 풍성해지나 구역당 도구 수가 적어 빈약해 보임.

## 결과 (예상)

- 프론트 도구 메타(`front/src/config/toolConfigs/`)에 구역 필드 추가, 사이드바가 현재 구역 스코프의 카테고리를 표시(+구역 전환 스위처).
- ⌘K는 구역과 무관하게 전역 검색 유지.
- 백엔드 무변경(표현 계층 원칙 — ADR-0017과 동일 노선). `getCategory()`도 그대로.
- 후속: v2 로드맵과 작업 분해는 `.scratch/v2-plan.md`.
