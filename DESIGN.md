# DESIGN.md — OnTool(온툴) 디자인 시스템

> UI를 만들거나 고치는 에이전트/개발자는 작업 전에 이 문서를 읽는다.
> 토큰의 원본은 `front/src/style.css`(shadcn-vue + Tailwind v4 `@theme`) — 이 문서와 코드가 어긋나면 코드가 진실이고, 이 문서를 갱신한다.
> 근거 결정: ADR-0015(Workbench 리디자인), ADR-0018(UX 개정), ADR-0023(v2 포털·구역).

## 0. 브랜드 (2026-07-17 확정)

**이름: OnTool / 온툴** — "온"(全, '모든'의 순우리말) + Tool. 모든 도구가 있는 곳.

| 맥락 | 표기 |
|------|------|
| 문장·제목·`<title>`·OG·문서 | `OnTool` |
| 로고 워드마크 (사이드바·대문 히어로) | `ontool` — 전부 소문자, **JetBrains Mono**, `on`은 `primary`(코발트)·`tool`은 `foreground`. 히어로에서는 끝에 터미널 커서(`_`) 깜빡임 허용(reduced-motion 존중) |
| 한글 병기·구어 | 온툴 |

- **파비콘**: 라운드 사각(radius 규율 준수) 코발트 배경 + 흰 소문자 `on` 모노그램(JetBrains Mono). 단일 SVG로 라이트/다크 겸용. 현 보라색 번개 `favicon.svg`는 임시물 — 교체 대상.
- **OG 이미지**: 그래파이트 배경 + `ontool` 워드마크 + 슬로건 1줄. 1200×630.
- **슬로건(확정, 2026-07-17): "모든 도구, 한 곳에"** — 히어로·OG 이미지·메타 description의 기본 카피. (보조 카피가 필요한 자리에는 "개발부터 생활까지" 톤 허용)
- 로고 심볼(그림 마크)은 만들지 않는다 — 워드마크+모노그램 체계로 충분(과한 브랜딩 금지).
- 도메인: 구매 보류 중(ADR-0023). 구매 시점에 ontool 계열 가용성 확인.

## 1. 정체성 — "Workbench"

**작업대.** 도구를 쓰러 온 사람이 3초 안에 일을 시작하는 화면. 화려함보다 밀도·정확성·즉각성.

- 뉴트럴 + 그래파이트 바탕, **액센트는 단 하나**(코발트). 색이 많은 화면은 이 사이트가 아니다.
- 모노스페이스(JetBrains Mono)는 **도구명·수치·라벨·코드 전용** 포인트. 본문에 쓰지 않는다.
- 라이트 = 웜 페이퍼 그레이 + 그래파이트 잉크. 다크 = 미드나잇 그래파이트(파랑 기미 살짝). **다크모드는 선택이 아니라 필수 지원.**
- 사이드바는 본문과 색 계열(hue)을 분리해 "프레임"으로 읽히게 한다.

## 2. 색 — 토큰으로만 쓴다

임의 hex/oklch 직접 사용 금지. Tailwind 유틸(`bg-background`, `text-muted-foreground`, `border-border`, `bg-primary` …)로만 접근한다.

| 토큰 | 라이트 | 다크 | 용도 |
|------|--------|------|------|
| `background` / `foreground` | `oklch(0.972 0.0035 85)` / `oklch(0.235 0.012 262)` | `oklch(0.185 0.008 262)` / `oklch(0.92 0.005 262)` | 페이지 바탕/본문 잉크 |
| `card` | `oklch(1 0 0)` | `oklch(0.225 0.009 262)` | 카드·패널 표면 (+`border-border`) |
| `primary` | `oklch(0.495 0.185 257)` | `oklch(0.74 0.14 257)` | **코발트 — 유일 액센트.** CTA·active·포커스 링 |
| `secondary` / `muted` / `accent` | 페이퍼 톤 계열 | 그래파이트 계열 | 보조 표면·비활성. accent는 hover 배경용이지 "색"이 아니다 |
| `muted-foreground` | `oklch(0.52 0.014 262)` | `oklch(0.67 0.012 262)` | 보조 텍스트·설명 |
| `destructive` | `oklch(0.577 0.245 27)` | `oklch(0.704 0.191 22)` | **파괴 동작·에러 전용.** 강조용 금지 |
| `border` / `input` / `ring` | `oklch(0.898 0.005 85)` / 〃 / =primary | `oklch(1 0 0 / 10%)` / 14% / =primary | 경계·입력 테두리·포커스 |
| `sidebar` 계열 | `oklch(0.9 0.006 260)` 등 | `oklch(0.11 0.005 262)` 등 | 사이드바 전용(본문과 hue 분리) |

규칙:
- 화면당 `primary` 액션은 원칙적으로 1개(실행/저장). 나머지는 secondary/ghost.
- 상태색을 늘리지 않는다 — 성공은 텍스트/아이콘으로, 실패만 `destructive`.
- 반투명 강조는 `color-mix(in oklab, var(--primary) 6%, transparent)` 패턴(FileUploader hover 참조).

### 2-1. v2 구역(Zone) 액센트 — 제안값, 구현 시 WCAG 대비 검증 필수

구역 컨텍스트(구역 홈·사이드바 active·구역 뱃지)에서 `--zone-accent`가 primary 역할을 대체한다. 명도·채도 규율은 기존 코발트와 동일하게 유지한다(라이트 L≈0.50/C≤0.19, 다크 L≈0.74/C≤0.14).

| 구역 | 라이트 | 다크 | 성격 |
|------|--------|------|------|
| `/dev` 개발자 도구 | `oklch(0.495 0.185 257)` (기존 코발트) | `oklch(0.74 0.14 257)` | 기준색 승계 |
| `/files` 파일·문서 | `oklch(0.50 0.12 175)` | `oklch(0.75 0.11 175)` | 틸 — 차분한 작업실 |
| `/life` 생활 도구 | `oklch(0.55 0.13 65)` | `oklch(0.78 0.12 70)` | 앰버 — 따뜻한 일상 |
| `/fun` 재미·게임 | `oklch(0.52 0.17 315)` | `oklch(0.75 0.14 315)` | 바이올렛 — 놀이 |

구역 액센트는 **컨텍스트 표시용**이다. 페이지 안의 실제 CTA·포커스는 여전히 하나의 액센트 원칙을 따른다(구역 안에서는 zone-accent가 그 하나).

## 3. 타이포그래피

| 토큰 | 값 | 용도 |
|------|-----|------|
| `--font-sans` | Pretendard Variable → system | 본문·제목 전부 |
| `--font-mono` | JetBrains Mono Variable | 도구명, 수치, 라벨, 코드, 결과 텍스트 **전용** |

- 새 폰트 추가 금지. 웨이트로 위계를 만든다(제목 semibold, 본문 regular).
- 크기는 Tailwind 스케일: 본문 `text-sm`(14px) 기본, 설명 `text-xs`, 페이지 제목 `text-xl~2xl`. 대문 히어로만 예외적 대형 허용.
- 숫자가 정렬되는 곳(카운터·통계·순번)은 mono + `tabular-nums`.

## 4. 형태·간격·모션

- 라운드: `--radius: 0.5rem` 기준 4단계(sm/md/lg/xl = ±4px). 알약형·완전 원형 남발 금지(뱃지·아바타만).
- 간격: Tailwind 4px 스케일. 카드 내부 `p-4~6`, 섹션 사이 `gap-6~8`. 콘텐츠 최대 폭 `max-w-[1440px]`.
- 그림자: 낮게. 표면 구분은 그림자보다 `border-border`가 우선.
- 모션: 목적 있는 것만 — 진입 스태거(`.stagger-in`, 0.35s ease-out, 항목당 30ms 지연), hover/focus 트랜지션 0.15s. **`prefers-reduced-motion` 존중 필수**(기존 패턴 유지). 장식성 무한 애니메이션 금지.

## 5. 레이아웃

- 데스크톱: 사이드바 `w-[280px]` 상시 표시 + 본문. 도구 페이지는 2패널 grid(입력 | 결과), 작업영역 `min-h ~40vh`.
- **모바일(v2 필수)**: 사이드바는 드로어로 전환, 2패널은 세로 스택(입력 위, 결과 아래), 터치 타겟 최소 44×44px. 기준 뷰포트 375px에서 깨지지 않아야 한다.
- 대문(`/`)은 사이드바 없는 전용 레이아웃 — 유일하게 "마케팅 페이지" 문법이 허용되는 곳. 단 팔레트는 이 문서의 토큰 안에서(§8 예외 조항).

## 6. 컴포넌트 규칙

- **shadcn-vue 우선.** 새 UI가 필요하면 ① 기존 공용 컴포넌트 재사용(FileUploader·ResultViewer·StructuredResultView·FileResultPanel·PdfThumbnail) → ② shadcn-vue 조합 → ③ 그래도 없으면 신규 제작 순서.
- 로딩: 스피너 + 현재 상태 문구("대기 중… 앞에 N개"처럼 구체적으로). 빈 화면 금지.
- 에러: `destructive` 톤 + **구체적 한국어 메시지**("요청이 너무 많습니다. 1분 후 다시 시도해 주세요" 스타일 — "오류가 발생했습니다" 금지).
- UI 문자열은 전부 한국어(i18n 미도입).

## 7. 인터랙션 품질 — Vercel Web Interface Guidelines 준수

UI 작업의 리뷰 기준은 `web-design-guidelines` 스킬(Vercel WIG)이다. 최소 체크:

- 모든 플로우 키보드 조작 가능, 포커스는 `:focus-visible` 링(ring 토큰).
- 입력은 `<form>`으로 감싸 Enter 제출, label 클릭 시 포커스, 적절한 `type`/`required`.
- 아이콘 단독 버튼은 `aria-label` 필수.
- 이미지 `max-width:100%`, 넓은 콘텐츠(표·코드)는 자체 `overflow-x-auto` — 페이지 가로 스크롤 금지.

## 8. Don't (금지 목록)

- 토큰 밖 임의 색상 / 화면당 액센트 2개 이상 / destructive의 강조 목적 사용
- 본문 모노스페이스, 새 폰트 도입
- 라이트 전용 구현(다크 미검증 머지 금지), reduced-motion 무시
- 그라데이션·글래스모피즘·네온 — **예외: 대문 히어로 한정**, 그것도 토큰 팔레트(코발트·그래파이트·구역 액센트) 조합으로만
- 페이지 가로 스크롤, 44px 미만 터치 타겟(모바일)
