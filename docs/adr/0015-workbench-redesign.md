# ADR-0015 Workbench 리디자인 — 다크모드·모바일 드로어·작업대 확장

## 상태
확정 — 단, §4(모바일 드로어)·§5(테마 순환 버튼)는 이후 **ADR-0018**에서 개정됨(2026-07). 프레젠테이션 레이어 통합은 **ADR-0017** 참조.

## 배경

초기 프론트는 shadcn-vue 기본 토큰(인디고 프라이머리, 흰 배경, 기본 radius)을 거의 그대로 사용해 "어떤 제품에 놓아도 똑같이 생긴" 화면이었다. 사용자가 직접 브라우저로 화면을 검토한 뒤 다음 문제를 지적했다:

1. 작업 영역(입력·출력 패널)이 뷰포트 대비 지나치게 좁고, 반대로 댓글·통계 같은 부차 요소가 화면 하단을 크게 차지했다.
2. 모바일 폭(400px)에서 사이드바가 고정 320px로 남아 본문이 사실상 파손됐다.
3. 홈 화면이 44개 도구를 위계 없이 나열해 무엇부터 봐야 할지 알 수 없었다.
4. `.dark` CSS 토큰은 존재했지만 `color-scheme: light only`로 도달 불가능했다.
5. 카드 아이콘 색이 카테고리별로 무작위 분포해 정보를 전달하지 못했다.

## 검토한 선택지

| 방식                                    | 설명               | 문제점                                                                  |
|---------------------------------------|------------------|----------------------------------------------------------------------|
| A. 세부 디테일만 조정 (여백·타이포만 손질)            | 최소 변경            | 근본 문제(위계 부재·모바일 파손·다크모드 도달불가) 미해결                                    |
| B. 기본 UI를 상단 호버 메가메뉴로 전면 교체           | 트렌디해 보이는 대안      | 도구 앱에서 세로 공간을 낭비하고, 발견성·접근성이 사이드바보다 떨어짐. 실제 주 내비게이션인 ⌘K 팔레트와도 궁합이 나쁨 |
| C. 사이드바+⌘K 골격 유지, 토큰/레이아웃/작업대를 전면 재작업 | 뼈대는 유지하고 표면만 재설계 | —                                                                    |

## 결정

**C. 사이드바 골격 유지 + Workbench 토큰 체계로 전면 재작업**

1. **디자인 정체성**: 뉴트럴+그래파이트 팔레트, 액센트 1색(코발트), JetBrains Mono를 도구명·수치·라벨 전용으로 사용(본문은 Pretendard 유지).
2. **다크모드**: 하드코딩 색상(`slate-*`, `zinc-*`, `indigo-*`)을 전부 시맨틱 토큰(`bg-card`, `text-muted-foreground` 등)으로 마이그레이션. `useTheme` 컴포저블(라이트/다크/시스템, localStorage `devtoolbox-theme`) + 사이드바 토글. `index.html` 인라인 스크립트로 FOUC 방지.
3. **작업대 확장**: 도구 페이지 컨테이너를 `max-w-4xl` → `max-w-[1440px]`로, 입출력 패널을 뷰포트 높이 기준(`min-h ~40vh`)으로 확장. 통계·좋아요는 헤더로 축소, 댓글은 접이식(개수 배지)으로 하단 이동.
4. **모바일**: lg 미만에서 사이드바를 오버레이 드로어로 전환 + 모바일 톱바(햄버거·로고·검색). 단, 세부 모바일 최적화는 공식 요구사항이 아니라고 확인됨(기본 동작 보장 수준에서 마무리).
5. **홈 위계**: 즐겨찾기/최근 사용/카테고리별 섹션으로 재구성. 카드 아이콘은 뉴트럴 톤으로 통일하고, 카테고리 구분은 색이 아닌 섹션 헤더가 담당.
6. **자동 실행**: Light 도구에 600ms 디바운스 자동 실행 추가(외부 HTTP를 호출하는 `html-fetch` 등은 제외해 키 입력마다 외부 요청이 나가지 않게 함), 빈 입력 상태에 "예시" 버튼 추가.

## 이유

- 메가메뉴(안 B)는 도구 앱의 진짜 주 내비게이션(⌘K)과 역할이 겹치고 세로 공간을 낭비해 기각.
- 다크모드는 이미 토큰이 존재했으므로 "완성"에 가까운 저비용 고가치 작업.
- 자동 실행은 이 사이트의 목적(빠른 1회성 변환)에 맞고, 외부 호출 도구만 예외 처리해 부작용을 통제.

## 결과

- 44개였던 도구 표시 목록은 이후 ADR-0017의 통합으로 34개로 줄었다(본 ADR 시점에는 카테고리 10→8 재편만 있었고, 도구 통폐합은 별도 ADR).
- 브라우저 E2E로 라이트/다크 전환, 모바일 드로어, 홈 섹션 진입을 확인했다.
