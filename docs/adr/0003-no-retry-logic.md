# ADR-0003 자동 재시도 제거

## 상태
확정

## 배경
무거운 Job 처리 실패 시 자동 재시도(최대 2회) 도입을 검토했다. `retry_count` 컬럼과 워커 내 재시도 로직이 필요하다.

## 결정
**자동 재시도 없음.** 실패 시 즉시 FAILED 처리.

## 이유

파일 변환 실패의 주요 원인을 따져보면 재시도가 실질적 효과가 없다.

| 실패 원인 | 재시도 효과 |
|------|------|
| 깨진 / 지원 안 되는 파일 | 없음 — 같은 파일, 같은 실패 |
| PDFBox / ImageIO 처리 오류 | 없음 — 같은 코드, 같은 실패 |
| OOM (메모리 부족) | 불확실 |
| 파일 저장 I/O 오류 (로컬 디스크) | 대부분 디스크 부족/권한 — 재시도해도 같은 실패 |

대부분의 실패는 입력 파일 문제다. 업로드 시점에 파일 형식·크기를 검증하면 실패 가능성이 더 낮아진다.

추가 복잡도(retry_count 컬럼, 재시도 간격 로직, 재시도 후 상태 관리)에 비해 실제 효과가 없다.

## 결과
- Job 테이블에서 `retry_count` 컬럼 제거
- 워커: 예외 발생 시 즉시 FAILED 처리
- 사용자가 재처리를 원하면 파일을 다시 업로드
